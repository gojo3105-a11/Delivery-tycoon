## 주차별 마일스톤

| 주차 | 마일스톤 | 완료 조건 |
|------|---------|---------|
| **Week 1** | 프로젝트 초기 설정 | Unity 프로젝트 생성, 폴더 구조, 네임스페이스, 첫 씬 |
| **Week 2** | 코어 시스템 | ServiceLocator, DataManager, SaveManager 동작 |
| **Week 3** | 경제 시스템 | EconomyManager, EconomyCalculator 테스트 통과 |
| **Week 4** | 기본 게임루프 | 주문 생성 → 배달 완료 → 코인 획득 → 시설 업그레이드 루프 작동 |
| **Week 5** | 오프라인 보상 | 앱 종료 후 재진입 시 오프라인 보상 팝업 정상 표시 |
| **Week 6** | 채용 시스템 | 코인/젬으로 택배원 채용, 등급 뽑기 확률 정상 동작 |
| **Week 7** | 광고 + IAP | AdMob 보상형 광고 실제 작동, IAP 테스트 구매 성공 |
| **Week 8** | UI 폴리시 | 임시 UI → 실제 아트 에셋 교체, 애니메이션 추가 |
| **Week 9** | 분석 + 크래시 | Firebase Analytics 이벤트 20개 정상 전송, Crashlytics 연동 |
| **Week 10** | 밸런스 조정 | 첫 구역 잠금 해제에 30분 이하, D1 잔존율 목표 40% |
| **Week 11** | QA 전수 테스트 | QA 체크리스트 전 항목 Pass |
| **Week 12** | 소프트 런칭 | 특정 국가 스토어 배포 완료 |

## 포스트 MVP (소프트 런칭 이후)

### Phase 2 (Month 4~5): 콘텐츠 확장

| 기능 | 설명 |
|------|------|
| 이벤트 시스템 완성 | 2주 단위 시즌 이벤트, 랭킹 보드 |
| 신화 캐릭터 추가 | 최초 신화 캐릭터 3종 추가 |
| 구역 9~10 추가 | 글로벌 허브, 미래 도시 구역 |
| 시즌 패스 | 유료/무료 트랙, 독점 보상 |
| 차량 강화 시스템 | 업그레이드 부품 소모 강화 |

### Phase 3 (Month 6~7): 소셜 및 라이브옵스

| 기능 | 설명 |
|------|------|
| 길드/협동 | 공동 배달 목표 달성 |
| 월드 랭킹 | 전 세계 플레이어 비교 |
| 콜라보 이벤트 | IP 협업 이벤트 택배원 |
| Firebase A/B 테스트 | 온보딩 최적화, 가격 실험 |
| OTA 에셋 업데이트 | 앱 업데이트 없이 신규 캐릭터 추가 |

### Phase 4 (Month 8+): 수익화 최적화

| 기능 | 설명 |
|------|------|
| 개인화 오퍼링 | 유저 행동 기반 맞춤 패키지 |
| 구독 모델 개선 | VIP 티어 확대 |
| 한정 판매 최적화 | 데이터 기반 가격 및 번들 조정 |
